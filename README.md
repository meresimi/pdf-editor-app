# PDF Editor — React + Capacitor

A mobile-first PDF annotation app built with React, Vite, and Capacitor.  
Annotate PDFs with moveable, styled text boxes and download the result.

---

## Features

- Open any PDF from device storage
- Place moveable text boxes anywhere on a page
- Number-controlled font size (tap the number to type, or use ±)
- Font family, bold, italic, color controls
- Copy / paste text boxes
- Zoom in/out on pages
- Multi-page support with thumbnail sidebar
- Save annotated PDF back to device (Android Downloads via Share sheet)
- Both toolbar strips are **horizontally swipeable** on mobile

## Build & run on Android (local)

### Prerequisites
- Node.js 20+
- Android Studio with SDK 34
- Java 17

```bash
# 1. Build the web app
npm run build

# 2. Add Android platform (first time only)
npx cap add android

# 3. Sync web assets into Android project
npx cap sync android

# 4. Open in Android Studio
npx cap open android
# Then: Build → Build APK(s) in Android Studio
```

## Build APK via GitHub Actions

Every push to `main` automatically builds a **debug APK**.

### Step 1 — Push the Android folder

After running `npx cap add android` locally, commit the `android/` folder:

```bash
git add android/
git commit -m "chore: add Android platform"
git push origin main
```

The workflow at `.github/workflows/build-android.yml` will trigger.  
Download the APK from **Actions → your run → Artifacts → pdf-editor-debug**.

---

### Step 2 — Signed release APK (on git tag)

Create a keystore once:

```bash
keytool -genkey -v \
  -keystore release.keystore \
  -alias pdf-editor \
  -keyalg RSA -keysize 2048 \
  -validity 10000
```

Add three **GitHub Secrets** (Settings → Secrets → Actions):

| Secret name        | Value                                        |
|--------------------|----------------------------------------------|
| `KEYSTORE_BASE64`  | `base64 -w 0 release.keystore` output        |
| `KEYSTORE_PASSWORD`| Password you set when creating the keystore  |
| `KEY_ALIAS`        | `pdf-editor`                                 |
| `KEY_PASSWORD`     | Key password (often same as keystore)        |

Then tag a release:

```bash
git tag v1.0.0
git push origin v1.0.0
```

GitHub Actions will build a signed APK and attach it to a GitHub Release automatically.

## Capacitor plugins used

| Plugin                   | Purpose                          |
|--------------------------|----------------------------------|
| `@capacitor/filesystem`  | Write PDF to device Documents    |
| `@capacitor/share`       | Native share/save sheet          |

These fall back to a browser `<a download>` when running in the browser.

---

## Changing the App ID

Edit `capacitor.config.json`:

```json
{
  "appId": "com.yourcompany.pdfeditor",
  "appName": "PDF Editor"
}
```

Then run `npx cap sync android` again.
